使用 Epoll 实现的 Echo 回响服务

Posted on 2025-03-11 In Demo
参考刘丹冰《深入理解 Go 语言》一书实现

Linux 的 IO 多路复用中，所谓的复用，意思是复用一个线程或者进程，多个网络或者文件 IO 就绪的时候无需开多个进程或者线程来处理。

select、poll、epoll 都是同步阻塞型 IO ，一句话概括三者的区别： - select - 内核帮忙轮询处理应用进程要求的文件描述符 IO 是否就绪，一旦存在任何就绪 IO ，调用 select 的应用进程从阻塞休眠中唤醒，自行遍历所有文件描述符查找可读写 fd（文件描述符）。 - poll - select 最大只支持 1024 个 fd 的监控，poll 解除了这个限制，其他并无二样 - epoll - 相比于前两者服务更加周到，不仅使得调用 epoll 的应用进程在存在就绪 IO 的时候从阻塞休眠中唤醒，并且返回所有就绪事件的数组给应用进程，每个事件中包含了该事件的状态（EPOLLIN 或者 EPOLLOUT），以及该事件关联的 fd ，由应用进程自行处理每个就绪的 fd IO

使用 epoll 的回响服务器实现

使用 epoll 的两个最重要的系统接口： - epoll_create(int fd_size) ：创建一个 epoll 监听入口的句柄，这是和内核 epoll 红黑树唯一联系的符号 - epoll_ctl(epfd, FLAG, *event) ：对指定的 epoll 句柄执行的操作，包括添加监听事件和移除监听事件，以及修改某个事件的监听状态（对哪种事件类型更感兴趣）

服务端

参考《深入理解 Go 语言》- 刘丹冰 中对 Linux IO 多路复用一节的介绍，主要是利用 epoll 来替代多进程管理连接，如果每来一个客户端连接就创建一个进程，那么服务器的资源很快就会消耗殆尽，因为一个进程占用的资源实在是太多了；多线程服务器也许是一个不错的选择，不过这和利用内核提供的 epoll 机制也差不多，哪个事件/线程就绪就去处理哪个，可能多线程服务器还比 epoll 在客户端的响应上要好一点。

另外在照着书上的代码实现的时候，发现有一些地方写的不太好，比如对用户输入输出的提示之类的。以及获取用户连接的 ip 地址和端口，实际上这里有一个点需要注意，那就是接受客户端连接的 accept 调用要求的第三个参数 addr_len ，发现 C 很多参数的传递都是利用指针来传递，可能是为了减少参数拷贝的性能开销，第三个参数一定要设置成 sizeof client_addr 之后，后续对该地址的读写才能有效！否则读出来的数据都是 零值 。

epoll 多路复用单进程单线程服务器的思路是，最开始加入到 epoll 事件红黑树中的只有监听套接字 fd ，只监听这个 fd 上的可读状态，直到第一个客户端连接到来，accept 之后系统内核创建了该 tcp 连接所需的套接字文件并将该套接字的 fd 通过函数 accept 的返回值返回给用户应用进程 ，此时由用户进程负责保管并维护该 fd ，当前的实现做法是将该 fd 放入内核提供的 epoll 红黑树中，由 epoll 来负责帮我们完成事件的监听，之后每来一个监听套接字 fd 上的可读事件，我们就 accept 创建新的客户端连接并将新的连接放入 epoll 红黑树中。

这段代码的意思是，客户端连接使用上升沿触发（Edge Trigger），而不是水平触发，关于上升沿和水平其实是数字电路信号中的知识，一个电压信号从低电压上升为高电压的瞬间成为上升沿，同理高电压变为低电压的瞬间是下降沿，一个上升沿在当前的预警中就是一个客户端的可读事件到来，如果这个数据一直没有被处理则电压水平保持高电平，如果选择了水平触发，则只要是高电平的信号，都会被 epoll 捕获到一直放在就绪队列中，当前的实现选择了边沿触发，也就是当前 epoll_wait 返回的就绪事件中，如果某个事件未处理，则下一次 epoll_wait 结束阻塞之后它不会被返回，尽管他有未处理的数据，直到该 fd 下一次数据到来的电压变化被 epoll 捕获，才会再次被放入就绪队列被用户进程看到进而处理，这是一种合理的选择，有些数据我们并不急着处理，而是处理刚刚到来的数据。

## read 和 fgets

read 是内核提供的一个比较低级的接口，它的功能非常简单，用户进程指定读取流多少字节，它就返回多少字节（除非读到 EOF 或者发生错误），它不会在读取完毕之后额外帮用户剔除换行符 '\n' 或者帮用户添加字符串的结束标识符 '\0' ，大概是因为 read 专精于字节流的处理，和字符串的概念是不相关的

fgets() 则在检测到换行符 '\n' 之后结束读取流，并将缓冲的下一个位置设置为字符串结束标识符 '\0' ，用户需要自己决定需不需要覆盖掉换行符，fgets() 把换行符也会读取进缓冲中。
